# ALICE Grounding Report

## Live Grounding Inputs

- Organizations and tenant context
- Portal metrics
- Automation events
- Automation OS registry
- Runtime execution counts
- Automation failure counts
- Enterprise cloud state where configured

## Changes

- ALICE operational answers now include Automation OS registry count, active automation count, failed automation count, and execution count.
- ALICE insight generation now reports automation coverage from `automation_registry` rather than generic static posture.

## Remaining Gap

Action-taking governance is not yet autonomous; ALICE can observe and recommend, while operator approval remains required.
